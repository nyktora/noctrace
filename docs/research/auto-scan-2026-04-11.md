## 1. New JSONL Fields Noctrace Should Parse

### `system` init record — new fields
```typescript
{
  agents?: string[],           // active agent names
  betas?: string[],            // active beta headers
  slash_commands?: string[],
  skills?: string[],           // loaded skill names
  plugins?: { name: string; path: string }[]
}
```
**Opportunity:** Enrich the Context Startup flyout with agents, skills, and plugins loaded at session start.

### `system` `compact_boundary` — new metadata
```typescript
{
  compact_metadata: {
    trigger: "manual" | "auto",   // distinguishes /compact from auto
    pre_tokens: number            // token count before compaction
  }
}
```
**Opportunity:** Show compaction trigger type and token delta in the health bar.

### `assistant` message — typed error field
```typescript
{
  error?: "authentication_failed" | "billing_error" | "rate_limit" |
          "invalid_request" | "server_error" | "max_output_tokens" | "unknown"
}
```
**Opportunity:** Replace pattern-matching API error detection with structured field reading.

### `result` message — new fields
```typescript
{
  duration_api_ms: number,         // time in API calls only
  modelUsage: { [model: string]: ModelUsage },  // per-model breakdown
  stop_reason: "end_turn" | "max_tokens" | "refusal" | null,
  permission_denials: SDKPermissionDenial[],
  structured_output?: unknown      // present when --json-schema used
}
```
**Opportunity:** Show API-vs-total time ratio, per-model cost breakdown, permission denial summary.

### Hook observability records in JSONL
New `system` subtypes when hooks are active:
- `hook_started` / `hook_progress` / `hook_response`

Also in JSONL transcript (not just hooks):
- `progress` rows with `data.type === "hook_progress"` — which hook is running
- `progress` rows with `data.type === "agent_progress"` — nested subagent tool calls with `data.agentId`

**Opportunity:** Show hook execution as waterfall rows; show in-progress subagent activity before completion.

### `parent_tool_use_id` field
Now explicitly typed as `string | null` on both `AssistantMessage` and `UserMessage`. This is the canonical mechanism for linking sub-agent messages to parent tool_use blocks.

### `isMeta` renamed to `isSynthetic`
SDK types now use `isSynthetic` for synthetic/injected user messages. Noctrace parser uses `isMeta` — may need to handle both.

## 2. New Hook Events (28+ total, up from ~4)

### Session lifecycle
- `SessionStart` — includes `source: "startup" | "resume" | "clear" | "compact"` and `agent_type`
- `SessionEnd`
- `InstructionsLoaded` — fires when CLAUDE.md or rules files load

### Per-turn
- `UserPromptSubmit` — can inject `additionalContext` or `sessionTitle`
- `Stop` / `StopFailure`

### Tool execution
- `PreToolUse` — now supports `updatedInput` (mutate tool inputs) and `permissionDecision: "defer"`
- `PostToolUse` — now supports `updatedMCPToolOutput` (rewrite MCP results)
- `PostToolUseFailure` — new, fires on tool crash/timeout (distinct from error results)
- `PermissionRequest` / `PermissionDenied`

### Sub-agent events (NEW)
- `SubagentStart` — payload: `{ agent_id, agent_type }` — fires BEFORE subagent completes
- `SubagentStop` — payload: `{ agent_id, agent_type, agent_transcript_path, last_assistant_message }`

### Team/task events (NEW)
- `TeammateIdle`, `TaskCreated`, `TaskCompleted`

### File/config events (NEW)
- `FileChanged`, `CwdChanged`, `ConfigChange`, `WorktreeCreate`, `WorktreeRemove`

### Context events (NEW)
- `PreCompact` / `PostCompact`

### MCP events (NEW)
- `Elicitation` / `ElicitationResult`
- `Notification`

### All hook payloads now include
```json
{
  "session_id": "...",
  "transcript_path": "/path/to/transcript.jsonl",
  "agent_id": "agent-abc123",
  "agent_type": "Explore"
}
```
**Opportunity:** `SubagentStart` lets noctrace show in-progress subagent rows in real time. `PreCompact`/`PostCompact` enables precise compaction markers. `agent_id` on all payloads enables per-subagent hook attribution.

## 3. New Documentation Pages

### Agent SDK (entirely new section)
- `/en/agent-sdk/observability` — OpenTelemetry tracing/metrics built into Claude Code
- `/en/agent-sdk/sessions` — Session management API: `listSessions()`, `getSessionMessages()`, `renameSession()`, `tagSession()`
- `/en/agent-sdk/cost-tracking` — Token/cost from message stream
- `/en/agent-sdk/streaming-output` — Real-time structured streaming

### OTel Observability
Three signal types exported:
- **Metrics**: tokens, cost, sessions, LOC, tool decisions, active time
- **Log events**: prompts, API requests, tool results, API errors
- **Traces** (beta): spans per interaction, model request, tool call, hook

OTel span names:
- `claude_code.interaction`, `claude_code.llm_request`, `claude_code.tool`
- `claude_code.tool.blocked_on_user`, `claude_code.tool.execution`, `claude_code.hook`

Notable OTel event attributes:
- `speed: "fast" | "normal"` — fast mode indicator
- `event.sequence` — monotonic counter for ordering when timestamps tie
- `git_commit_id` — on successful git commit tool calls
- `mcp_server_scope` — MCP server attribution
- `prompt.id` — correlates all events from a single user prompt

### Other new pages
- `/en/agent-teams` — multi-session agent team coordination (separate JSONL files per teammate)
- `/en/channels` — push events into running sessions from Telegram, Discord, webhooks
- `/en/remote-control` — continue local sessions from any device
- `/en/context-window` — interactive context window visualization reference
- `/en/whats-new/` — weekly changelog (2026-w13, 2026-w14 latest)

## 4. New CLI Flags

| Flag | Relevance |
|---|---|
| `--include-hook-events` | Include hook lifecycle events in stream-json output |
| `--debug-file <path>` | Write debug logs to file |
| `--debug "api,hooks"` | Category-filtered debug mode |
| `--json-schema <schema>` | Validated JSON output; adds `structured_output` to result |
| `--agents <json>` | Define subagents dynamically per session |
| `--fork-session` | New session ID when resuming |
| `--effort low\|medium\|high\|max` | Reasoning effort level; affects token usage |
| `--max-budget-usd <amount>` | Cap total spend; result gets `error_max_budget_usd` |
| `--bare` | Minimal mode — skips hooks, plugins, MCP |
| `--teleport` | Resume web session in local terminal |
| `cleanupPeriodDays` setting | Session files older than N days deleted at startup (default 30) |

## 5. Competing Projects & Ecosystem

### Direct competitors
| Project | Stars | Stack | Approach | Key differentiator |
|---|---|---|---|---|
| **claude-code-trace** | 175 | Rust + React | JSONL viewer | Native desktop GUI |
| **abtop** | 156 | Rust + TUI | Real-time monitor | htop-style terminal UI |
| **agents-observe** | 404 | Node + React | Hook-driven dashboard | Claude Code plugin marketplace distribution |
| **claumon** | Active | Go | Background service | Memory browser with relationship graph |
| **ccusage** | High adoption | Node CLI | Cost tracker | De-facto community cost calculator |
| **tail-claude** | 130 | Go + Bubble Tea | TUI transcript viewer | Terminal-native |

### agents-observe (simple10) — v0.8.1 (2026-04-07)
Most feature-rich competitor. Notable patterns:
- Dual-format parser handling both hook events AND JSONL transcripts
- Parses `agent_progress` rows for real-time subagent activity
- Extracts `subAgentName` and `subAgentDescription` from PreToolUse before call completes
- Stores `ownerAgentId` (from `raw.agent_id`) on every hook event
- `agents` table with: `id`, `session_id`, `parent_agent_id`, `name`, `description`, `agent_type`, `agent_class`, `transcript_path`
- Distributed as Claude Code plugin: `claude plugin marketplace add simple10/agents-observe`

### contextspectre (ppiankov) — stable, v0.48.0
Go CLI for JSONL editing and analysis (not visualization):
- Reads both Claude Code CLI and Claude for Mac sessions
- 9 cleanup operations: removes stale tool results, compaction summaries, thinking blocks
- `rewire` command: synthesizes missing `tool_result` entries for orphaned `tool_use` blocks
- `launch` command: checkpoint + fix + clean + `claude --resume`
- File locking (`flock`/`LockFileEx`) on all write paths
- Metrics: Cost Per Decision, Turns To Convergence, Context Drift Rate, scope drift, $/hour velocity

### disler/claude-code-hooks-multi-agent-observability — 1,359 stars
Reference implementation for hook-based observability:
- All 12 hook events via Python → HTTP POST → Bun + SQLite → Vue 3
- Uses `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1` (undocumented experimental feature)
- `statusLine` hook renders live status in Claude Code terminal UI
- Feb 2026 update: added `SubagentStart` support and `task`/`team` tool names

### OTel bridge projects
Growing pattern of JSONL → OpenTelemetry conversion:
- `otel-agent-trace` — JSONL to Jaeger traces
- `claude-code-otel-exporter` — JSONL to OTel metrics (updated 2026-04-10)
- `claude-code-phoenix-otel` — session traces to Phoenix/Arize
- `claude-code-otel-collector-google-cloud` — OTel on Cloud Run

## 6. Noctrace-Specific Opportunities (Prioritized)

### High-value, low-effort
1. **Parse `compact_metadata`** — show trigger type + pre-compaction token count on health bar compaction lines
2. **Parse `assistant.error` typed field** — replace pattern matching for API errors
3. **Parse `result` enrichments** — `duration_api_ms`, `modelUsage`, `stop_reason`
4. **Parse `system` init enrichments** — `agents[]`, `skills[]`, `plugins[]` for Context Startup flyout
5. **Handle `SubagentStart` hook** — show in-progress subagent rows before they complete

### Medium-effort, high-value
6. **Parse `agent_progress` JSONL rows** — show real-time subagent tool calls streaming through main session
7. **Parse `hook_started`/`hook_progress`/`hook_response`** — show hook execution as waterfall rows
8. **`event.sequence` ordering** — use monotonic counter when timestamps tie
9. **`speed` field** — badge for fast mode requests
10. **`parent_tool_use_id`** — canonical parent-child linking (supplement current `agentId` approach)

### Larger features
11. **OTel export** — bridge to Grafana/Jaeger/Datadog for enterprise users
12. **Agent teams topology view** — separate JSONL files per teammate, coordinate via hook events
13. **Plugin marketplace distribution** — `claude plugin marketplace add nyktora/noctrace`
14. **`--effort` level display** — show reasoning effort level affecting token usage

### Competitive gaps to watch
- agents-observe's plugin marketplace distribution model is gaining traction
- ccusage's per-model, per-billing-window cost breakdown is more detailed than noctrace's
- abtop's real-time terminal monitoring complements noctrace's browser-based retrospective view
- The OTel bridge pattern lets tools integrate with existing enterprise observability stacks

## 7. Experimental / Undocumented

- `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1` + `"teammateMode": "in-process"` — in-process agent teams (vs. subprocess subagents). JSONL behavior under this mode may differ.
- `statusLine` hook — renders live status bar in Claude Code terminal UI. Separate integration surface.
- `TRACEPARENT` W3C context injected into Bash subprocesses — enables end-to-end distributed tracing.
- `cleanupPeriodDays` setting — session files auto-deleted after N days (default 30). Affects noctrace's session picker if old sessions disappear.
