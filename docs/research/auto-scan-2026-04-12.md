## Docker/Remote Environment Research (completed 2026-04-11)

Key findings for running noctrace with containerized Claude Code:

### Approach A: Bind-mount (zero noctrace changes)
```bash
docker run -v ~/.claude:/root/.claude my-claude-container
```
Sessions written in the container appear at the standard host path. Noctrace's chokidar watcher picks them up automatically.

### Approach B: HTTP hooks to host
```json
{
  "hooks": {
    "PostToolUse": [{
      "matcher": "*",
      "hooks": [{ "type": "http", "url": "http://host.docker.internal:4117/api/hooks" }]
    }]
  }
}
```
Docker Desktop: `host.docker.internal` works out of the box. Linux: add `--add-host=host.docker.internal:host-gateway`.

### Approach C: OTEL endpoint
Configure `OTEL_EXPORTER_OTLP_ENDPOINT=http://host.docker.internal:4317` in the container. Noctrace already has `GET /api/session/:slug/:id/otlp` for export; a receiver endpoint would complete the loop.

### Key facts
- Config dir env var: `CLAUDE_CONFIG_DIR` (not `CLAUDE_HOME`)
- JSONL path: `$CLAUDE_CONFIG_DIR/projects/<encoded-cwd>/<session-id>.jsonl`
- Hook failures are non-blocking (noctrace being down won't break Claude Code)
- `persistSession: false` in Agent SDK skips disk writes entirely
